Package Configuration Fails to Apply

When configuring a sub-option under an existing, previously configured option in Buildroot, the new sub-configuration may not take effect.

Example (Python):

For instance, after previously configuring and compiling Python3, adding a new sub-option such as python-aiocoap under external python modules may not be applied upon recompiling the filesystem.

Solution:

1. Re-make the package directly. This will incorporate the changes. (Not recommended, as repackaging is still required to include the modifications into the final filesystem image.)

When you need to compile a package separately, you can use the make xxx-rebuild directive, using python3 as an example:

Execute at OK3568-linux-source/build root

2. Simply remove the package and then recompile it (Recommended). This directly packs the changes into the final image.

A full Linux 5.10 kernel compilation does not automatically rebuild the filesystem. Please first execute: ./build.sh buildroot
