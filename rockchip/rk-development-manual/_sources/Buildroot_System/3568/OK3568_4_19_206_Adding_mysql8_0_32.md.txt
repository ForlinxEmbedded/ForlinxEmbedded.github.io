MySQL-related files; please download before reading this document. All files mentioned in the text are included in the compressed package.

1\. Library Preparation

Place the mysql-lib folder in any directory on the development board, then copy the library files to the /usr/lib/ directory.

**<font style="color:rgb(0,0,0);">\[root@ok3568:/home/forlinx/mysql-lib]#</font>** <font style="color:rgb(68,114,196);">cp ./\* /usr/lib/</font>

Then create soft links for these files so that the MySQL service can find and use them

**\[root@ok3568:/home/forlinx/mysql-lib]#** <font style="color:rgb(68,114,196);">ln -s /usr/lib/libaio.so.1.0.1 /usr/lib/libaio.so.1</font>

**\[root@ok3568:/home/forlinx/mysql-lib]#** <font style="color:rgb(68,114,196);">ln -s /usr/lib/libnuma.so /usr/lib/libnuma.so.1</font>

**\[root@ok3568:/home/forlinx/mysql-lib]#** <font style="color:rgb(68,114,196);">ln -s /usr/lib/libncurses.so.6.0 /usr/lib/libncurses.so.5</font>

**\[root@ok3568:/home/forlinx/mysql-lib]#** <font style="color:rgb(68,114,196);">ln -s /usr/lib/libncurses.so.6.0 /usr/lib/libtinfo.so.5</font>

2\. Configuration file preparation

Place the my. cnf in the board's/etc directory

3\. mysql Installation

Unzip mysql-8.0.32-linux-glibc2.17-aarch64.tar

**\[root@ok3568:/home/forlinx/mysql-lib]#**\*\* \*\*<font style="color:rgb(68,114,196);">tar -xvf mysql-8.0.32-linux-glibc2.17-aarch64.tar</font>

Place the extracted folder in the/usr/local directory and rename it mysql8

**\[root@ok3568:/home/forlinx/mysql-lib]#**\*\* \*\*\*\*  
\*\*<font style="color:rgb(68,114,196);">mv mysql-8.0.32-linux-glibc2.17-aarch64 /usr/local/mysql8</font>

4\. mysql Initialization

Switch to/usr/local/mysql8 path for execution

**\[root@ok3568:/usr/local/mysql8]#**\*\* \*\*<font style="color:rgb(68,114,196);">./bin/mysqld --initialize</font>

**Note: If the initialization fails, you need to clear the files in the mysql8/data directory and reinitialize.**

Enable mysql.service

**\[root@ok3568:/usr/local/mysql8]#**\*\* \*\*<font style="color:rgb(68,114,196);">./support-files/mysql.server start</font>

![Image](./images/OK3568_4_19_206_Adding_mysql8_0_32/1719740442914_5695b1a7_5f0f_423d_a193_ab310a94180c.png)

**Note: If initialization completes without errors but mysql.server start fails, the system date on the development board may be incorrect. Use the date command to set it to the current time, then delete the contents of the data folder, reinitialize, and start again.**

5\. Connect to the Local Database

Check the initialization password described in the data/error.log file in the current directory:

**\[root@ok3568:/usr/local/mysql8]#**\*\* \*\*<font style="color:rgb(68,114,196);">vi data/error.log</font>

![Image](./images/OK3568_4_19_206_Adding_mysql8_0_32/1719740447338_6ad4d336_6ae7_4ea9_ae1e_4d2e27cd5380.png)

6\. Log in to the local database:

**\[root@ok3568:/usr/local/mysql8]#**\*\* \*\*<font style="color:rgb(68,114,196);">./bin/mysql -h 127.0.0.1 -u root -p</font>

// Enter the password found in data/error.log when prompted. After successful login, MySQL is ready for use.

// Once logged in, you can change the root login password to 123456 using: alter user 'root'@'localhost' identified by '123456';
