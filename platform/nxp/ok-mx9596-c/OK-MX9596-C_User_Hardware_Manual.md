# 01_NXP i.MX95简介

i.MX 95应用处理器提供先进的图形和视频核心，强大的视觉和机器学习加速，高效的CPU性能加上实时处理和先进的安全，集成的EdgeLock®安全领域，以支持节能的边缘计算。

i.MX 95应用处理器集成了多达6个Arm Cortex®-A55核，内置的Arm Cortex®-M33，支持功能安全，它可以配置为一个安全岛。优化工业、物联网和汽车设备的性能和功率效率。i.MX 95处理器采用了NXP创新的能源弹性架构。

i.MX 95应用处理器提供了一套针对汽车、工业和商业物联网市场的丰富外设。这是EdgeVerse™智能边缘解决方案组合的一部分。i.MX 95系列将提供商业、工业、扩展工业和汽车水平的资格认证，并得到NXP的产品寿命计划的支持。

## 3.5 **OK-MX9596-C底板说明**

### 3.5.1 底板电源
底板电源为直流12V，由DC-005插座（P9）引入。VCC_12V电源经过静电防护器件、自恢复保险丝、防反接二极管后，给核心板供电；后经MP8765降为VDD_5V，这个电源受核心板控制，为底板供电；后经过MP2147降为VCC_3V3，这个电源受核心板控制，为底板供电；通过RT9013降为VCC_1V8，这个电源受核心板控制，为底板供电；同时VCC_12V电源分别经过2个MP2307降为PCIE1_3V3与PCIE2_3V3，这两个电源不受核心板控制，分别为2个PCIE供电。

MP8765的EN引脚由核心板输出的VDD_3V3拉高，从而保证VCC_12V经过MP8765输出VCC_5V，进而输出VCC_3V3、VCC_1V8给底板各电路提供电源。

此部分电路是保证了核心板先上电，底板后上电，以防闩锁效应的发生损坏CPU。

### 3.5.2 开关机键、复位与休眠按键

### **3.5.3 Boot配置**
i.MX95有多种烧写和启动方式，在系统上电或复位后，通过读取系统启动配置引脚的状态，选择不同的烧写和启动方式。开发板使用UART1_TXD/BT_MODE0、UART2_TXD/BT_MODE1、SAI1_TXFS/BT_MODE2、SAI1_TXD/BT_MODE3作为系统启动配置引脚。由于与相关的功能有复用关系，需要保证实际的功能电路对 启动项电路不能产生影响，建议增加模拟切换开关来确保启动项可以不受其他电路影响。

i.MX95 的启动项包含了16种状态：

| BOOT_MODE[3:0] | BOOT CORE | Function |
| --- | --- | --- |
| 0000 | Cortex-A55 | Boot from Internal Fuses |
| 0001 | Cortex-A55 | Serial Downloader(USB1) |
| 0010 | Cortex-A55 | USDHC1 8-bit eMMC 5.1 |
| 0011 | Cortex-A55 | USDHC2 4-bit SD3.0 |
| 0100 | Cortex-A55 | FlexSPI Serial NOR |
| 0101 | Cortex-A55 | FlexSPI Serial NAND 2K page |
| 0110 | Cortex-A55 | FlexSPI Serial NAND 4K page |
| 0111 | Cortex-A55 | Reserved |
| 1000 | Cortex-M33 | Boot from Internal Fuses |
| 1001 | Cortex-M33 | Serial Downloader(USB1) |
| 1010 | Cortex-M33 | USDHC1 8-bit eMMC 5.1 |
| 1011 | Cortex-M33 | USDHC2 4-bit SD3.0 |
| 1100 | Cortex-M33 | FlexSPI Serial NOR |
| 1101 | Cortex-M33 | FlexSPI Serial NAND 2K page |
| 1110 | Cortex-M33 | FlexSPI Serial NAND 4K page |
| 1111 | Cortex-M33 | Reserved |


目前核心板支持的启动项有M33的OTG烧写、M33的EMMC启动和M33的TF卡启动。

### 3.5.4 调试串口
调试串口采用的是TYPE-C的形式，使用XR21V1414芯片将Cortex-A55、 Cortex-M33、Cortex-M7的三个调试串口转换成一组USB总线，方便使用和同时调试三个内核。 

UART1是Cortex-A55 调试串口，2线，3.3V电平；UART2是Cortex-M33调试串口，2线，3.3V 电平；UART3是Cortex-M7调试串口，2线，3.3V 电平。

核心板调试串口信号经过缓冲芯片之后再经过转接芯片，最后来到TYPE-C接口，保证了在底板未上电的时候插入调试串口，都不会对核心板和底板产生漏电问题。

### 3.5.6 音频
开发板上具有一路HP/MIC接口、一路板载MIC接口和两路SPAKER接口。HP/MIC接口使用四段式耳机座引出，同时可以实现录音和放音；板载MIC可以直接实现录音功能；两路SPAKER接口通过2.54间距白色端子座引出。耳机具有插拔检测的功能， 当耳机插入时，放音将会自动切换到耳机通道上来，SPAKER将不再放音。

音频芯片NAU88C22内部自带的D类功放输出端，可驱动两只8Ω喇叭，最高输出功率为1W，如 果需要外接更大的功放，只能从耳机插座获取信号，不能从喇叭接口获取信号。

 注意：

喇叭的功率来自D类功放，不是传统的模拟功放； 

每个插座接一个喇叭，切记不可共用喇叭线，也不能把喇叭信号接到地线上；

### 3.5.9 TYPE-C接口
在底板中将USB2.0连接到 Type-C 接口P32用作烧录功能口，同时将USB3.0接入Type-C接口，此TYPE-C接口支持正反插和主从转换。

当在从模式下，主设备（如电脑）通过电缆插入TYPE-C座时，TYPEC_PWR_EN信号将底板给TYPE-C 接口的供电切断，保证从主设备而来的5V与开发板的5V二者不发生冲突；当在主模式下，从设备通过电缆插入TYPE-C座时，TYPEC_PWR_EN信号将底板供电打开，确保给从设备的正常供电。
